# 智能 OnCall Agent 简历项目说明

## 简历版本

智能 OnCall Agent 运维问答系统

项目简介：

基于 Spring Boot 3.2、Java 17、DashScope 和 Milvus 实现的智能运维问答系统。系统支持知识库文档上传、文本切片、Embedding 向量化、Milvus 相似度召回和 RAG 增强问答，并通过阿里云 ECS + Docker Compose + Nginx 部署到公网。

技术栈：

Java 17、Spring Boot、DashScope、Milvus、Docker Compose、Nginx、systemd、REST API。

工作内容：

- 搭建 Spring Boot 后端服务，提供智能问答、知识库上传、RAG 检索和知识库管理接口。
- 接入 DashScope 文本模型和 Embedding 模型，实现基于知识库片段的 RAG 回答。
- 使用 Milvus 存储向量数据，并实现 topK 相似文档召回。
- 使用 Docker Compose 管理 Milvus、MinIO、Etcd 依赖服务。
- 使用 Nginx 反向代理和 systemd 服务管理，将项目部署到阿里云 ECS 并开放公网访问。
- 为公网 `/chat` 接口增加 Token 鉴权，避免接口被未授权调用。
- 编写公网验证脚本和敏感信息检查脚本，验证服务可用性、接口鉴权和 GitHub 仓库安全性。

项目亮点：

- 不只是本地 Demo，而是可以通过公网地址访问和演示。
- 覆盖从后端接口、AI 模型调用、向量数据库到云部署的完整链路。
- 具备可测试性：可以围绕接口可用性、鉴权、异常输入、RAG 召回质量和部署健康状态设计测试用例。
- 真实 Key 和 Token 通过环境变量配置，公开仓库中只保留占位符和示例文件。

## 面试讲解版本

这个项目是一个面向运维和客服场景的智能问答系统。用户可以上传 Markdown 或 TXT 格式的知识库文档，系统会对文档进行切片，然后调用 DashScope Embedding 模型生成向量并存入 Milvus。当用户提问时，系统先从 Milvus 召回相似知识片段，再把召回内容和用户问题一起交给大模型生成答案。

我在项目里主要做了三类工作：

第一类是后端接口，包括知识库上传、知识库列表、RAG 检索、普通问答和标准 `/chat` 接口。

第二类是部署，把项目从本地服务变成公网可访问项目。部署时使用 Docker Compose 管理 Milvus 相关服务，使用 systemd 管理 Java 进程，使用 Nginx 把公网 80 端口反向代理到本地 9900 端口。

第三类是测试和交付。我整理了公网验证脚本，可以检查首页、知识库接口、RAG 检索接口和 `/chat` 鉴权逻辑；也整理了敏感信息检查脚本，避免把真实 API Key 或 Token 提交到 GitHub。

如果面试官问“你这个项目怎么测试”，可以回答：

- 冒烟测试：先检查首页、知识库列表、RAG 检索、`/chat` 鉴权是否可用。
- 接口测试：分别验证上传文件、查询知识库、召回知识片段、普通问答和标准问答接口。
- 异常测试：传空问题、超长问题、错误 Token、无 Token、不支持的文件格式。
- 安全测试：检查公网只开放 80/443，不直接暴露 Milvus、MinIO 和 Java 内部端口；检查真实 Key 不进入 Git。
- 回归测试：每次修改部署脚本或后端接口后，重新执行公网验证脚本，确认核心链路没有坏。
