# RAG 测试与评估方案

## 测试目标

验证智能体是否能基于知识库回答客服、售后、运维、日志、图片工单类问题，而不是完全依赖大模型自由发挥。

## 测试方法

1. 先上传 `contest-materials/knowledge-base` 下的 Markdown 文件。
2. 使用 `rag_test_cases.csv` 中的问题逐条提问。
3. 检查回答是否包含预期关键词。
4. 对需要知识库依据的问题，检查回答是否与知识库内容一致。
5. 对图片问题，上传包含错误码、订单号或告警信息的截图，检查智能体是否先提取图片信息再给出处理建议。

## 评分建议

检索命中率：回答包含对应文档中的关键处理步骤，记为命中。

答案准确率：没有编造支付状态、订单状态、客户信息，且能给出正确升级路径，记为准确。

幻觉抑制：信息不足时会追问订单号、支付流水号、traceId 或截图，而不是直接下结论。

多轮一致性：在同一个 session_id 下连续追问，能记住前面的问题背景。

## 演示用问题

1. 客户说已经支付成功但订单没生成，我该怎么回复？
2. 如果日志里有 `signature_verify_failed`，可能是什么原因？
3. ServiceUnavailable 告警应该如何处理？
4. P0 和 P1 工单响应时间分别是多少？
5. 上传一张支付或告警截图，问智能体“这张图里有哪些关键排障信息？”
