# 优化建议清单

## 1. 已完成优化

- 接入 DashScope 文本模型，支持真实问答。
- 接入 DashScope 视觉模型兼容接口，支持图片字段扩展。
- 构建客服、售后、运维类知识库。
- 支持 Markdown/TXT 上传、分片、Embedding 和 Milvus 存储。
- 新增 RAG 调试接口 `/api/rag/search`。
- 前端展示引用来源，提升可解释性。
- 新增知识库管理页面，展示已上传文件。
- 新增比赛标准 `/chat` 接口。
- 增加接口 Token 鉴权。
- 完成阿里云 ECS 公网部署。
- 使用 Nginx 80 端口反向代理 Spring Boot。
- 使用 systemd 管理 Java 服务并设置开机自启。

## 2. 赛前建议优先完成

### 2.1 补充云端截图

建议新增公网运行截图，证明项目不只是在本地运行：

- `http://121.40.90.107/` 首页。
- 公网问答和引用来源。
- 知识库管理页面。
- 服务器 Docker 容器 healthy 状态。

### 2.2 扩充测试集

继续扩充 `contest-materials/test-set/rag_test_cases.csv`，覆盖：

- 支付订单异常。
- 退款/售后政策。
- ServiceUnavailable 告警。
- 慢响应排查。
- 日志关键词分析。
- 图片工单信息抽取。

### 2.3 增加自动化评测

建议写一个脚本批量读取 `rag_test_cases.csv`，自动调用：

- `/api/rag/search`
- `/chat`

并输出命中来源、回答摘要和是否通过人工预期。

### 2.4 增加管理接口鉴权

当前 `/chat` 已有 Token 鉴权。为了公网部署更安全，建议后续给这些接口也加管理权限：

- `/api/upload`
- `/api/knowledge/files`
- `/api/rag/search`
- `/milvus/health`

比赛演示阶段可以保留开放，正式上线建议限制访问。

## 4. 当前提交建议

当前版本已经满足演示和初赛提交的主体要求：

- 有可访问系统。
- 有标准接口。
- 有知识库投喂。
- 有 RAG 能力。
- 有智能体工具调用设计。
- 有公网部署。
- 有技术文档、接口说明、运行说明、验证报告和演示脚本。

后续重点不再是“能不能跑”，而是“材料是否讲得清楚、截图是否有说服力、测试用例是否覆盖充分”。
