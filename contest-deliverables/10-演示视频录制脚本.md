# 演示视频录制脚本

本文件记录当前产品功能演示视频的录制流程与复现命令。最终视频文件为：

```text
contest-deliverables/OnCallAgent_Function_Demo.mp4
```

视频为无音频的 1080p 产品实操素材，后续可继续与答辩 PPT、旁白或字幕进行剪辑组合。

## 1. 视频内容

| 时间段 | 画面内容 | 覆盖能力 |
| --- | --- | --- |
| 0:00-1:10 | 打开系统首页，依次展示左侧智能问答、知识库管理、接口调试入口 | 产品首页与系统结构 |
| 1:10-2:10 | 打开知识库管理面板，展示 7 份已入库资料、上传资料入口和入库状态 | 知识库管理、资料投喂 |
| 2:10-2:35 | 输入“支付订单超时如何排查？”，等待系统完成回答 | RAG 问答请求 |
| 2:35-2:55 | 从本次回答顶部开始，稳定向下滚动展示完整回答和引用来源 | 回答全量展示、TopK=3 来源追溯 |
| 2:55-3:07 | 打开接口调试页面，分别执行无 Token 请求和带 Token 请求 | `/chat` 接口、Token 鉴权 |

## 2. RAG 完整展示规则

录制脚本不会以发送问题后的自动滚动结果作为正式展示起点。脚本会先等待回答完整生成，再定位最后一条带引用来源的 assistant 消息，并将聊天滚动容器移动到该回答顶部。

RAG 展示逻辑：

1. 输入问题：`支付订单超时如何排查？`
2. 点击发送。
3. 等待本次 assistant 回答生成完成，并确认引用来源卡片数量达到 3。
4. 将 `[data-testid="chat-scroll-container"]` 滚动到最后一条 `[data-testid="assistant-message"]` 顶部。
5. 在回答顶部停留 8 秒。
6. 以约 300px/s 的速度向下自然滚动。
7. 滚动经过排查步骤、处理建议、总结内容和引用来源。
8. 在引用来源处短暂停留，确保 TopK=3 可见。

为保证脚本稳定定位，前端已补充以下 DOM 标识：

```html
<div data-testid="chat-scroll-container">
  <div data-testid="user-message">...</div>
  <div data-testid="assistant-message">...</div>
</div>
```

这些标识只服务于自动化录制和测试，不改变业务逻辑和页面视觉。

## 3. 启动命令

在项目根目录启动本地服务：

```powershell
$env:CONTEST_API_TOKEN = "demo-recording-token"
mvn -gs maven-settings-central.xml -s maven-settings-central.xml spring-boot:run "-Dspring-boot.run.profiles=local" "-Dspring-boot.run.arguments=--server.port=9910"
```

浏览器访问地址：

```text
http://127.0.0.1:9910/
```

## 4. 录制命令

自动录制脚本：

```text
scripts/record_function_demo.py
```

执行命令：

```powershell
python scripts\record_function_demo.py --url http://127.0.0.1:9910/ --token demo-recording-token --output contest-deliverables\OnCallAgent_Function_Demo.mp4
```

脚本会启动一个临时 Chrome 实例，通过页面真实点击、输入、上传和接口请求完成录制。录制过程中不会显示 IDE、终端、开发者工具或真实密钥。
