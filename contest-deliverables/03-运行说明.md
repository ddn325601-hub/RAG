# 运行说明

## 1. 本地环境

建议环境：

- Windows 10/11
- Java 17 或更高版本
- Maven
- Docker Desktop
- WSL2
- DashScope API Key

项目根目录：

```text
C:\path\to\SuperBizAgent
```

## 2. 本地环境变量

项目通过环境变量读取 DashScope Key。

```powershell
[Environment]::SetEnvironmentVariable("DASHSCOPE_API_KEY", "你的真实 Key", "User")
```

重新打开 PowerShell 后检查：

```powershell
[Environment]::GetEnvironmentVariable("DASHSCOPE_API_KEY", "User")
```

注意：不要把真实 Key 写入 Git、Markdown、截图或公开材料。

## 3. 本地启动 Docker 与 Milvus

项目提供 `vector-database.yml`，会启动：

- `milvus-etcd`
- `milvus-minio`
- `milvus-standalone`
- `milvus-attu`

启动命令：

```powershell
docker compose -f vector-database.yml up -d
```

检查命令：

```powershell
docker ps
```

正常状态应看到 `milvus-standalone` 和 `milvus-minio` 为 healthy。

## 4. 本地启动项目

推荐使用本地启动脚本：

```powershell
.\start-local.ps1
```

脚本会：

1. 检查 `DASHSCOPE_API_KEY`。
2. 启动 Milvus 相关容器。
3. 使用 `application-local.yml` 启动 Spring Boot。
4. 开启本地 mock 日志和 mock 监控，便于演示。

启动成功后访问：

```text
http://127.0.0.1:9900/
```

## 5. 云服务器部署方式

当前项目已部署在阿里云 ECS：

- 系统：Ubuntu 22.04 64 位
- 公网 IP：`121.40.90.107`
- Web 入口：`http://121.40.90.107/`
- Nginx：监听 80 端口，反向代理到本机 `127.0.0.1:9900`
- Java 服务：`super-biz-agent.service`
- 项目目录：`/opt/SuperBizAgent-Java`
- Milvus 数据目录：`/opt/super-biz-agent-data`

服务器服务管理命令：

```bash
systemctl status super-biz-agent
systemctl restart super-biz-agent
journalctl -u super-biz-agent -f
docker compose -f /opt/SuperBizAgent-Java/deploy/vector-database-prod.yml ps
```

服务器环境变量文件：

```text
/etc/super-biz-agent.env
```

该文件保存 `DASHSCOPE_API_KEY`、`JAVA_OPTS`、`CONTEST_API_TOKEN` 等敏感配置，权限应保持为 `600`，不要提交到 Git。

## 6. 投喂知识库

知识库文件目录：

```text
contest-materials\knowledge-base
```

本地批量上传示例：

```powershell
$baseUrl = "http://127.0.0.1:9900"
$kbDir = ".\contest-materials\knowledge-base"
Get-ChildItem -Path $kbDir -Filter *.md | ForEach-Object {
  curl.exe -F "file=@$($_.FullName)" "$baseUrl/api/upload"
}
```

公网批量上传示例：

```powershell
$baseUrl = "http://121.40.90.107"
$kbDir = ".\contest-materials\knowledge-base"
Get-ChildItem -Path $kbDir -Filter *.md | ForEach-Object {
  curl.exe -F "file=@$($_.FullName)" "$baseUrl/api/upload"
}
```

上传成功后，文件会复制到 `uploads/`，并自动生成向量写入 Milvus。

## 8. 常见问题

### Docker Desktop 一直转圈

建议确认：

- Docker Desktop 已使用 WSL2 backend。
- WSL 已更新。
- `com.docker.service` 正常运行。
- 没有其他虚拟化工具冲突。

### 项目提示 API Key 未设置

说明当前进程没有读取到 `DASHSCOPE_API_KEY`。本地可重新打开终端，或临时设置：

```powershell
$env:DASHSCOPE_API_KEY = [Environment]::GetEnvironmentVariable("DASHSCOPE_API_KEY", "User")
```

服务器上检查：

```bash
grep -E 'DASHSCOPE_API_KEY|CONTEST_API_TOKEN|JAVA_OPTS' /etc/super-biz-agent.env
systemctl restart super-biz-agent
```

检查时不要把完整 Key 截图或公开。

### RAG 检索没有结果

检查：

- Milvus 容器是否 healthy。
- `/milvus/health` 是否返回 `biz` collection。
- 知识库文件是否已经上传。
- `DASHSCOPE_API_KEY` 是否有 Embedding 模型调用权限。

### 视觉模型不可用

当前视觉模型配置在 `application-local.yml`：

```yaml
contest:
  vision:
    model: qwen3.7-plus
    endpoint: https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions
```

如果阿里云模型权限或模型名变化，需要同步修改该配置。
