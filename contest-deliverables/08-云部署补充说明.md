# 云部署补充说明

## 1. 部署目标

项目已从本地演示版本扩展为公网可访问版本，便于比赛评审、答辩演示或团队成员远程体验。

- 公网访问地址：`http://121.40.90.107/`
- 比赛接口地址：`POST http://121.40.90.107/chat`
- 服务器系统：Ubuntu 22.04 64 位
- 部署目录：`/opt/SuperBizAgent-Java`

## 2. 服务器组件

| 组件 | 作用 |
| --- | --- |
| Nginx | 监听公网 80 端口，反向代理到 Spring Boot |
| Spring Boot | 承载前端页面、比赛接口、RAG 和 Agent 逻辑 |
| Docker | 运行向量数据库依赖 |
| Milvus | 存储知识库向量 |
| Etcd | Milvus 元数据依赖 |
| MinIO | Milvus 对象存储依赖 |
| systemd | 管理 Java 服务开机自启 |

## 3. 服务管理命令

```bash
systemctl status super-biz-agent
systemctl restart super-biz-agent
journalctl -u super-biz-agent -f
```

向量数据库检查：

```bash
docker compose -f /opt/SuperBizAgent-Java/deploy/vector-database-prod.yml ps
```

Nginx 检查：

```bash
nginx -t
systemctl reload nginx
```

## 4. 敏感配置

服务器敏感配置位于：

```text
/etc/super-biz-agent.env
```

包含：

- `DASHSCOPE_API_KEY`
- `CONTEST_API_TOKEN`
- `JAVA_OPTS`
- `SPRING_PROFILES_ACTIVE`

注意：

- 不要把该文件提交到 Git。
- 不要在截图、论文、答辩 PPT 中展示完整 API Key。
- 如果 Token 泄露，可修改 `CONTEST_API_TOKEN` 后重启服务。

## 5. 公网验证命令

```powershell
$env:CONTEST_API_TOKEN = "演示 Token"
powershell.exe -NoProfile -ExecutionPolicy Bypass `
  -File .\contest-deliverables\api-examples\smoke-test.ps1 `
  -BaseUrl "http://121.40.90.107"
```
