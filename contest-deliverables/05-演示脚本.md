# 演示脚本

适合 3 到 5 分钟项目答辩或视频录制使用。

## 1. 开场说明

大家好，我们的项目是“智能 OnCall Agent”，面向客服、售后和运维场景，解决一线人员在故障排查、用户问答和知识检索中响应慢、依赖人工经验强、资料分散的问题。

项目已经完成本地运行和阿里云 ECS 公网部署，演示地址为：

```text
http://121.40.90.107/
```

## 2. 核心能力概览

本项目具备四类核心能力：

1. 真实大模型问答：接入 DashScope 模型，不是静态模板回复。
2. RAG 知识库：支持 Markdown/TXT 资料上传、分片、Embedding、写入 Milvus，并在回答中引用来源。
3. 智能体工具调用：具备内部文档检索、日志查询、指标查询和时间工具等 OnCall 辅助能力。
4. 多模态扩展：接口支持图片字段，视觉模型可用于截图类工单分析。

## 3. 页面演示流程

### 步骤 1：打开公网页面

访问：

```text
http://121.40.90.107/
```

说明页面中有聊天输入区、知识库入口和引用来源展示区域。

### 步骤 2：展示知识库管理

点击页面右上角或侧边的“知识库”入口，展示已投喂资料：

- 支付订单排查手册
- 售后政策说明
- 运维告警处理流程
- 日志与指标说明
- 图片工单分诊说明
- 常见 FAQ
- 产品支持总览

讲解重点：项目不是只靠大模型自由发挥，而是通过 RAG 结合企业资料回答。

### 步骤 3：演示 RAG 问答

输入：

```text
支付订单超时应该怎么排查？
```

预期回答应包含：

- 检查支付回调是否到达。
- 检查订单服务是否处理。
- 检查 MQ 是否堆积。
- 检查风控、库存、数据库等下游依赖。
- 页面下方展示引用来源，例如 `payment_order_runbook.md`。

讲解重点：回答能追溯到知识库来源，具备可解释性。

### 步骤 4：演示运维告警场景

输入：

```text
ServiceUnavailable 告警出现后，第一步应该排查什么？
```

预期回答应结合告警处理流程，给出 5xx、服务实例、日志关键词、指标曲线等排查路径。

### 步骤 5：演示比赛接口

说明项目提供标准 `/chat` 接口：

```text
POST http://121.40.90.107/chat
Authorization: Bearer <CONTEST_API_TOKEN>
```

可以用 `contest-deliverables/api-examples/smoke-test.ps1` 做接口验证。

## 4. 技术架构讲解

可以按这个顺序说明：

1. 前端页面负责聊天、来源展示和知识库管理。
2. Spring Boot 提供 `/chat`、`/api/upload`、`/api/rag/search` 等接口。
3. 文档上传后进入分片服务，调用 DashScope Embedding 生成向量。
4. 向量写入 Milvus，问答时先召回知识片段，再交给智能体组织回答。
5. Nginx 将公网 80 端口流量代理到本机 Java 服务。

## 5. 收尾总结

本项目已经从本地 Demo 改造为可公网访问的智能 OnCall Agent，具备真实模型调用、RAG 知识库、引用来源、知识库管理和比赛标准接口。后续可以继续扩展真实企业工单数据、监控系统接入和自动化评测能力。
