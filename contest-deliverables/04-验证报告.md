## 5. 比赛 `/chat` 接口验证

请求：

```powershell
$baseUrl = "http://121.40.90.107"
$token = $env:CONTEST_API_TOKEN
$body = @{
  question = "支付订单超时应该怎么排查？"
  session_id = "public-smoke"
} | ConvertTo-Json -Compress

Invoke-RestMethod `
  -Method Post `
  -Uri "$baseUrl/chat" `
  -Headers @{ Authorization = "Bearer $token" } `
  -ContentType "application/json; charset=utf-8" `
  -Body $body
```

响应摘要：

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "answer": "结论：支付订单超时应按标准排查手册分四层定位——支付回调是否到达、订单服务是否处理、消息队列是否堆积、下游依赖是否响应慢...",
    "session_id": "public-smoke"
  }
}
```

回答内容正确引用了 `payment_order_runbook.md`，并给出支付流水、订单服务、MQ、风控/库存/数据库等排查路径。

结论：公网 `/chat` 接口可以真实调用大模型，并结合 RAG 知识库生成答案。
